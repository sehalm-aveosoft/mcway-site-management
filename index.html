Key Architectural Decisions
✓
Linkage Principle — relationships entered on the owning side, read-only on parent
✓
Verification never blocks — entries usable before verified; Verifier is accountability, not a gate
✓
Diesel offset — rented-with-fuel: we fuel it, deduct from vendor bill. No double counting.
✓
Stock from transactions — no manual stock entry. Stock emerges from inward/outward/production.
✓
One user = one scope — supervisor sees only their site, camp in-charge sees only their camp.
✓
Machine usage (ON/OFF/idle) deferred to Phase 2 — Machine field as placeholder in On-Site module.
